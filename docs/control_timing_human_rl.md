# 人体の制御タイミングと強化学習の step 周波数

神経科学・運動制御における「入力から指令までの間隔」と、MuJoCo 強化学習における環境 step の Hz の目安をまとめたメモです。数値は文献・慣習上の**典型値**であり、個人差・タスク・実装で変わります。

関連: [sim_human_comparison.md](sim_human_comparison.md)、[exp_001 README](../mujoco-sim/mujoco_rl_sim/experiments/exp_001_2joint_a2c/README.md)、[exp_002 README](../mujoco-sim/mujoco_rl_sim/experiments/exp_002_2joint_a2c/README.md)

---

## 1. 人体：脳が入力を受け取り指示を出す間隔

### 1.1 一言での目安

**一つの入力 → 一つの指示**として測ると、**数十 ms（反射）〜数百 ms（意識的な反応）**の幅がある。

体全体を「一定周期で制御している」と捉えると、実効的には **おおよそ 5〜20 Hz（50〜200 ms ごと）** でフィードバックを回している、という説明がよく使われる。

### 1.2 レイヤー別

| レイヤー | 内容 | 目安 |
|----------|------|------|
| 反射（脊髄中心） | 伸張反射など | 刺激から筋収縮まで **約 30〜50 ms** |
| 感覚の上行 | 固有感覚・触覚 | 比較的速い（**十数 ms オーダー**から） |
| | 視覚 | 網膜→視覚野まで **数十 ms**、意味判断はさらに遅い |
| | 聴覚 | 視覚より速い |
| 意識的な反応 | 単純反応（視覚） | **約 150〜250 ms**（訓練で **100 ms 台**も） |
| | 単純反応（聴覚） | **約 140〜180 ms** 前後が多い |
| | 選択反応 | **200〜400 ms 以上**になりやすい |

※ 反応時間には筋への下行・筋の力学も含まれるため、「脳だけの処理時間」ではない点に注意。

### 1.3 連続動作の制御ループ（フィードバック更新）

意識的な運動では固定クロックではなく、**予測（フィードフォワード）＋必要なときだけフィードバック**が基本。

| フィードバックの種類 | 目安の更新間隔 | おおよその周波数 |
|----------------------|----------------|------------------|
| 固有感覚・力覚 | 50〜100 ms | **10〜20 Hz** |
| 視覚による軌道修正 | 100〜200 ms | **5〜10 Hz** |
| ゆっくりした計画変更 | 数百 ms〜秒 | **1 Hz 未満** |

### 1.4 ロボット・シミュレーションとの違い（重要）

1. **固定クロックではない** — デジタル制御のように一定周期で必ず更新するわけではない。
2. **予測で遅れを埋める** — 小脳などの内部モデルにより、感覚が返る前に指令を出している区間が長い。
3. **指示は一発ではない** — 運動皮質・小脳・基底核・脊髄が並列・階層で、複数の時間スケールが同時に動く。

### 1.5 シミュレーション設計への直感（参考）

| レベル | `dt` の目安 |
|--------|-------------|
| 反射レベル | 0.02〜0.05 s（20〜50 ms） |
| オンライン修正 | 0.05〜0.2 s（5〜20 Hz） |
| タスク判断・方針変更 | 0.2 s 以上 |

人間は「1 ms ごとの厳密ループ」ではなく、**多層のループが重なっている**と捉えるのが近い。

## 2. 強化学習：環境 step は何 Hz にするか

### 2.1 結論（推奨の考え方）

強化学習で決めるべきは主に **ポリシーが新しい行動を出す頻度（制御 Hz）**。物理シミュレーションはそれより速く回すのが一般的。

| レイヤー | おすすめ | 理由 |
|----------|----------|------|
| **物理（`mj_step`）** | **200〜1000 Hz**（`dt` = 0.001〜0.005 s） | 接触・倒れの数値安定性 |
| **RL の 1 step（行動更新）** | **50〜100 Hz**（`dt` = 0.01〜0.02 s） | 歩行・バランス系の定番帯。サンプル効率と人体のオンライン修正に近い |
| **人体に寄せる実験** | **10〜20 Hz** | 生物学的には妥当だが、転倒系では遅く感じやすい |

**まず試すなら 50 Hz（0.02 s/step）** が無難。不安定なら **100 Hz**、サンプル不足・学習が遅いなら **20〜50 Hz** へ調整する、という流れが多い。

### 2.2 2 つの Hz を分ける

```
[ポリシー] --行動--> [同じ行動を N 回保持] --> [物理 mj_step × N]
     ↑ 50〜100 Hz                    ↑ 500 Hz 前後
```

- **物理 Hz**: シミュレーションの数値安定性（貫通・振動の抑制）
- **制御 Hz**: MDP の時間刻み（エージェントが判断し直す間隔）

人体の **10〜20 Hz** は「脳が軌道を直す」イメージに近いが、**脊髄反射や筋の応答はもっと速い**。転倒防止だけを人体 Hz に合わせると、制御が粗くなりすぎることがある。

### 2.3 本リポジトリの現状（exp_001 / exp_002）

`env.py` では **1 RL step = 1 回 `mj_step`**。XML に `timestep` 指定がないため MuJoCo 既定の **0.002 s（500 Hz）** で、物理と制御が同じレートになっている。

観測・報酬は **1 step あたりの変化** を前提にしている（例: `MAX_DX_PER_STEP = 0.05` [m/step]、`config.py` / `observation.py`）。Hz を変える場合は、これらのスケールと `GAMMA`・エピソード長をセットで見直す必要がある。

| 項目 | 現状の影響 |
|------|----------|
| 制御レート | 実質 **500 Hz** |
| `MAX_STEPS_PER_EPISODE = 3000` | 実時間 **約 6 s**（3000 × 0.002 s） |
| `ROLLOUT_STEPS = 512` | 実時間 **約 1 s** 分のロールアウト |
| `GAMMA = 0.99` | step が細かいほど、同じ秒数での「見通しステップ数」が増える |

### 2.4 2 関節・前進・直立タスク向けの目安

| 設定 | 長所 | 短所・注意 |
|------|------|------------|
| **500 Hz（現状）** | 細かい制御、物理は安定しやすい | ステップ数が多く、同じ `GAMMA` では実時間あたりの見通しが短く感じられる |
| **50 Hz（物理 500 Hz + frame skip 10）** | Locomotion RL でよくある構成 | `dx`・速度の正規化を per-step 定義に合わせて調整 |
| **20 Hz 以下** | サンプル効率（同更新回数で長い実時間） | 倒れるまでの修正回数が減り、学習が難化しやすい |

### 2.5 Hz を選ぶときのチェックリスト

1. **ダイナミクスの時間定数** — 倒れ始めから着地まで何 step か。制御がその過程で **5〜20 回**は修正できると学習しやすい（経験則）。
2. **アクチュエータ** — 毎 step で目標を大きく変えすぎるとジッタ。実機でも **50〜100 Hz** 指令が多い。
3. **`GAMMA` とエピソード長** — 制御 Hz を下げると、同じ `max_steps` では実時間が伸びる。
4. **サンプル効率** — 低 Hz は同じ更新回数でカバーする実時間が長くなる（良い面・悪い面あり）。
5. **人体との対応** — 生物学的リアリズム: **10〜20 Hz** / 学習・安定性: **50〜100 Hz** / 現状維持: **500 Hz**（細かいシミュレーションとして解釈）。

### 2.6 実務的な推奨（優先順）

1. **物理は 500 Hz 前後を維持**（`timestep=0.002` で問題なければそのまま）
2. **ポリシーだけ 50 Hz**（1 行動あたり 10 sub-steps）を第一候補
3. 立ち上がりが遅い・ぎこちない → **100 Hz**
4. サンプル不足・方策の振動 → **20〜50 Hz**（観測・報酬の per-step スケールを要調整）
5. 人体への ablation として **10〜20 Hz** を別実験にする

## 3. 人体 Hz と RL Hz の対応（まとめ）

| 観点 | 人体 | RL（本タスクの目安） |
|------|------|----------------------|
| オンライン軌道修正 | 5〜20 Hz（感覚モダリティ依存） | **50〜100 Hz**（学習のデフォルト候補） |
| 反射・筋応答 | 20〜50 ms 級 | 物理 **200〜1000 Hz** で別途表現 |
| 方針・目標の変更 | 1 Hz 未満〜数 Hz | 報酬設計・エピソード長で間接的に表現 |
| 固定周期 | なし（多層・予測混在） | **物理と制御を分離**するのが定石 |

Hz を変えるときは、`dx`・角速度・`MAX_*_PER_STEP`、`GAMMA`、`MAX_STEPS_PER_EPISODE` を**同じ実時間・同じ物理意味**になるよう揃える。
