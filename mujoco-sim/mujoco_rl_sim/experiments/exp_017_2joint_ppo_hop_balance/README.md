# exp_017: 報酬バランス調整（前進↓・進捗/押し出し/着地↑）

## 仮説

exp_010 final（~3.11 m）が現状ベスト。exp_011〜016 では追加 shaping・終了緩和・転移微調整を試したが、多くは u3500 以降で 1〜2.7 m に悪化。**新しい項を足さず**、既存報酬の重みだけを調整して、ステップ前進報酬への過依存（ダイブ型）を抑え、ホップ周期（押し出し→着地→進捗）を強化する。

## 変更（exp_010 比）

| 項目 | exp_010 | exp_017 |
|------|---------|---------|
| 前進ステップ報酬 `FORWARD_REWARD_SCALE` | 80 | **55** |
| 進捗 / 押し出し / 着地 | 12 / 0.25 / 0.55 | **18 / 0.45 / 0.75** |
| 飛翔膝過屈曲ペナルティ | 2.5 | **3.2** |
| 観測・終了・XML | 同左 | **同左** |

転移元: **exp_010 final**（`run_20260524_173035/final.pt`、~3.1 m）

## 学習

```bash
cd mujoco-sim
python train.py \
  --resume "../archive/exp_010_2joint_ppo_hop_progress/run_20260524_173035/final.pt" \
  --lr 2e-4 --num-updates 2000
```
