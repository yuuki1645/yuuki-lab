# exp_024 メモ

exp_023 からの差分: 姿勢ペナルティのヨーすり抜け対策（`lib/pose.py`）。

- `lean_fwd_body` … 前後傾き・後傾終了・飛翔前傾ゲート
- `heading_align` … 正面維持 shaping（終了には使わない）
- `tilt_horiz` … 水平面傾き（横倒れ含む）

## 学習結果の考察（2026-05）

前進は達成したが、人間のような「片脚ずつ持ち上げて歩く」 gait には収束していない。

- 足を大きく持ち上げず、小刻みに振動しながら **すり足（shuffle）** で +X 前進する
- 腰から上の構造がなく、上半身でバランスを取れない現形態では、遊脚を作るほど転倒リスクが増える
- 報酬設計（前進 + 直立 + 短い遊脚ペナルティ等）の下では、**すり足が局所最適**になった可能性が高い
- 当初目標は人間型歩行だったが、現プログラム・現モデルでの **実用的な最適解** に収束した、という解釈
